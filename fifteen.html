1 Grant SELECT privilege on EMP table to a user named USER1. (GRANT – BASIC SELECT)
Show Answer
GRANT SELECT ON EMP TO USER1;
2 Grant SELECT and INSERT privileges on EMP table to USER2. (GRANT – MULTIPLE PRIVILEGES)
Show Answer
GRANT SELECT, INSERT ON EMP TO USER2;
3 Grant all privileges on DEPT table to USER3. (GRANT – ALL PRIVILEGES)
Show Answer
GRANT ALL ON DEPT TO USER3;
4 Grant SELECT privilege on EMP table to USER4 with GRANT OPTION. (GRANT – WITH GRANT OPTION)
Show Answer
GRANT SELECT ON EMP TO USER4 WITH GRANT OPTION;
5 Grant UPDATE privilege only on SAL column of EMP table to USER5. (GRANT – COLUMN LEVEL)
Show Answer
GRANT UPDATE(SAL) ON EMP TO USER5;
6 Grant DELETE privilege on EMP table to USER6. (GRANT – DELETE)
Show Answer
GRANT DELETE ON EMP TO USER6;
7 Grant SELECT privilege on both EMP and DEPT tables to USER7. (GRANT – MULTIPLE TABLES)
Show Answer
GRANT SELECT ON EMP TO USER7; GRANT SELECT ON DEPT TO USER7;
8 Grant privileges on EMP table to PUBLIC. (GRANT – PUBLIC ACCESS)
Show Answer
GRANT SELECT ON EMP TO PUBLIC;

Show Answer
GRANT UPDATE, DELETE ON EMP TO USER9;

15 Revoke UPDATE privilege on SAL column from USER5. (REVOKE – COLUMN LEVEL)

-- Note: Most SQL dialects require revoking the privilege from the whole table REVOKE UPDATE ON EMP FROM USER5;

28 Update department location and rollback before committing. (ROLLBACK – DEPT TABLE)

29 Insert a new employee, update salary, and commit only after verifying results. (TRANSACTION – COMMIT CONTROL)
Show Answer
INSERT INTO EMP (EMPNO, ENAME, SAL) VALUES (8001, 'NEHA', 4500); UPDATE EMP SET SAL = SAL + 200 WHERE EMPNO = 8001; -- Verifying... SELECT * FROM EMP WHERE EMPNO = 8001; -- Verification successful COMMIT;
30 Insert multiple employees, rollback one operation using SAVEPOINT, then commit remaining. (TRANSACTION – SAVEPOINT + COMMIT)
Show Answer
INSERT INTO EMP (EMPNO, ENAME) VALUES (9001, 'AJAY'); SAVEPOINT s1; INSERT INTO EMP (EMPNO, ENAME) VALUES (9002, 'VIJAY'); ROLLBACK TO s1; COMMIT; -- Note: Only AJAY is saved in the database.
31 Create a transaction where some operations are committed and others are rolled back. (TRANSACTION – PARTIAL CONTROL)
Show Answer
INSERT INTO EMP (EMPNO, ENAME) VALUES (1001, 'ROHAN'); SAVEPOINT p1; UPDATE EMP SET SAL = 2500 WHERE EMPNO = 1001; ROLLBACK TO p1; COMMIT; -- Result: ROHAN is inserted, but his salary update is undone.
32 Demonstrate use of SAVEPOINT in EMP table operations. (TRANSACTION – SAVEPOINT)
Show Answer
UPDATE EMP SET SAL = SAL + 100; SAVEPOINT sp_after_raise; DELETE FROM EMP WHERE DEPTNO = 30; -- If deletion was a mistake: ROLLBACK TO sp_after_raise;
33 Perform update on EMP table and rollback to a specific SAVEPOINT. (TRANSACTION – ROLLBACK TO SAVEPOINT)
Show Answer
SAVEPOINT start_point; UPDATE EMP SET COMM = 500 WHERE JOB = 'SALESMAN'; SAVEPOINT point_2; UPDATE EMP SET SAL = SAL * 1.10; ROLLBACK TO point_2; -- Result: Salesman commissions are updated, but the 10% raise is rolled back.
34 Grant privileges to a user, perform DML operations, and commit changes. (GRANT + TRANSACTION)
Show Answer
-- Step 1: Grant Access GRANT INSERT, UPDATE ON EMP TO USER_A; -- Step 2: (Logged in as USER_A) INSERT INTO EMP (EMPNO, ENAME, SAL) VALUES (2002, 'ANJU', 3200); COMMIT;
35 Revoke privileges and verify access restrictions using EMP table. (REVOKE – PRACTICAL SCENARIO)
Show Answer
-- Step 1: Revoke Access REVOKE SELECT ON EMP FROM USER_B; -- Step 2: Verification (Logged in as USER_B) SELECT * FROM EMP; -- Result: "ORA-00942: table or view does not exist" -- (Privilege verification successful)
