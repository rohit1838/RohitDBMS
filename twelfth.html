1 What is the DECODE function in SQL?
Show Answer
The DECODE function provides if-then-else logic in SQL. It compares an expression to each search value one by one. If the expression matches a search value, it returns the corresponding result. **Syntax:** DECODE(column, search1, result1, search2, result2, ..., default)
2 Using DECODE, display employee name and department name based on DEPTNO (10='ACCOUNTING', 20='RESEARCH', 30='SALES', else 'OTHERS').
Show Answer
SELECT ENAME, DECODE(DEPTNO, 10, 'ACCOUNTING', 20, 'RESEARCH', 30, 'SALES', 'OTHERS') AS DNAME FROM EMP;
3 What is the CASE expression?
Show Answer
The CASE expression is a more flexible version of DECODE. It can handle complex conditions (like <, >, BETWEEN) and is ANSI SQL compliant (works across different databases). **Syntax:** CASE WHEN condition THEN result ELSE default END
4 Using CASE, display employee name and salary grade ('High' if SAL > 3000, 'Medium' if SAL > 1500, else 'Low').
Show Answer
SELECT ENAME, CASE WHEN SAL > 3000 THEN 'High' WHEN SAL > 1500 THEN 'Medium' ELSE 'Low' END AS SALARY_GRADE FROM EMP;
5 What is the difference between Simple CASE and Searched CASE?
Show Answer
- Simple CASE: Compares one expression to multiple values (like DECODE). Example: CASE DEPTNO WHEN 10 THEN 'A' END - Searched CASE: Uses Boolean expressions for each WHEN clause. Example: CASE WHEN SAL > 2000 THEN 'B' END
6 Using DECODE, calculate a bonus: 'CLERK' gets 500, 'SALESMAN' gets 1000, 'MANAGER' gets 1500, others 0.
Show Answer
SELECT ENAME, JOB, DECODE(JOB, 'CLERK', 500, 'SALESMAN', 1000, 'MANAGER', 1500, 0) AS BONUS FROM EMP;
7 Rewrite the previous Bonus question using a CASE expression.
Show Answer
SELECT ENAME, JOB, CASE JOB WHEN 'CLERK' THEN 500 WHEN 'SALESMAN' THEN 1000 WHEN 'MANAGER' THEN 1500 ELSE 0 END AS BONUS FROM EMP;
8 Display employee name and status: 'Senior' if hired before 1982, 'Junior' if hired after 1981.
Show Answer
SELECT ENAME, HIREDATE, CASE WHEN HIREDATE < TO_DATE('01-JAN-1982', 'DD-MON-YYYY') THEN 'Senior' ELSE 'Junior' END AS STATUS FROM EMP;
9 Can you nest a DECODE function inside another DECODE?
Show Answer
Yes, DECODE functions can be nested to handle more complex logic. **Example:** DECODE(DEPTNO, 10, DECODE(JOB, 'CLERK', 'A', 'B'), 'C')
10 Display employee name, job, and a 'Duty' column: 'Office' for CLERK/MANAGER, 'Field' for SALESMAN, 'Lab' for ANALYST.
